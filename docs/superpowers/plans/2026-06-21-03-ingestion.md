# Incremental engine + ingestion — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect scrapers with the storage: normalize `RawListing`→`Listing`, incrementally sync the database (new/changed/gone listings + price history) and orchestrate scraping runs per portal with error isolation and statistics (`ScrapeRun`).

**Architecture:** Pure **normalizer** (`RawListing`→`Listing`, content hash) + **incremental engine** (`IncrementalEngine.sync_source`) operating on repositories + **IngestionService** orchestrating: for each source `run_search` (Plan 2) → normalization → `sync_source` → saving `ScrapeRun`. Each source in its own transaction and its own `try` — failure/block of one portal does not break others.

**Tech Stack:** Python 3.14, SQLAlchemy 2.0 async, Pydantic. Tests: pytest + real Postgres+pgvector (testcontainers, `engine` fixture), fake fetcher + real HTML fixtures.

## Global Constraints

- Python 3.14; execution via **uv**. SQLAlchemy 2.0 async. Migrations only via Alembic.
- Normalizer is a **pure function** (`RawListing`→`Listing`), without I/O.
- TDD; `uv run ruff check .` must pass.
- `raw_hash` = SHA-256 of canonical representation of significant listing fields (not from `raw`).
- Change of `raw_hash` → field update + `embedding=None` (to be recomputed in Plan 4) + (if price changed) entry in `price_history`.
- Isolation per source: one portal cannot break the entire run; `ScraperBlocked` and exceptions caught per source and recorded in `ScrapeRun`.
- Marking "gone" only for a successfully completed run (not for blocked/failed) — so a block does not delete active listings.

## Data conventions

`SyncStats` (dataclass): `new: int`, `updated: int`, `unchanged: int`, `gone: int`.

`ScrapeRunStatus` (StrEnum): `SUCCESS`, `BLOCKED`, `FAILED`.

from decimal import Decimal

from realestate.ingestion.normalize import compute_raw_hash, to_listing
from realestate.models.enums import ListingStatus, MarketType
from realestate.scrapers.base import RawListing


def _raw(**kw):

                price=Decimal("600000"), area_m2=50.0, rooms=3, market="secondary",
                images=["b.jpg", "a.jpg"])
    base.update(kw)
    return RawListing(**base)


def test_to_listing_maps_fields_and_computes_ppm2():

    assert listing.raw_hash

def test_price_per_m2_none_when_no_price():
    listing = to_listing(_raw(price=None), now=datetime.now(UTC))
    assert listing.price_per_m2 is None


def test_raw_hash_stable_and_content_sensitive():
    h1 = compute_raw_hash(_raw())
    h2 = compute_raw_hash(_raw(images=["a.jpg", "b.jpg"]))  # different image order
    h3 = compute_raw_hash(_raw(price=Decimal("700000")))
    assert h1 == h2  # image order irrelevant (sorted)
    assert h1 != h3  # price change changes hash

### Task 4: Incremental engine (`IncrementalEngine.sync_source`)

**Files:**
- Create: `src/realestate/ingestion/incremental.py`
- Test: `tests/ingestion/test_incremental.py`

**Interfaces:**
- Consumes: `ListingRepository`, `Listing`, `PriceHistory`, `compute_raw_hash`-based `raw_hash` already set on `Listing` (from normalizer).
- Produces:
  - `@dataclass class SyncStats: new=0; updated=0; unchanged=0; gone=0`
  - `class IncrementalEngine(session: AsyncSession)`:
    - `async def sync_source(self, source_id: str, listings: list[Listing], *, now: datetime, mark_missing_gone: bool = True) -> SyncStats`
    - For each `incoming` (transient `Listing` from normalizer, same `source_id`):
      - `existing = repo.get_by_external(source_id, incoming.external_id)`
      - missing → `session.add(incoming)`; if `incoming.price is not None` → add `PriceHistory(price=incoming.price, observed_at=now)` linked to `incoming`; `new += 1`.
      - exists, `existing.raw_hash == incoming.raw_hash` → `existing.last_seen = now`; `unchanged += 1`.
      - exists, hash differs → copy mutable fields from `incoming` to `existing` (`title, price, price_per_m2, area_m2, rooms, floor, total_floors, city, district, street, lat, lon, market, description, images, posted_at, raw_hash`), `existing.last_seen = now`, `existing.status = ACTIVE` (reactivation when returned), `existing.embedding = None`; if `incoming.price is not None and incoming.price != old_price` → add `PriceHistory(price=incoming.price, observed_at=now, listing=existing)`; `updated += 1`.
      - collect `seen_ids`.
    - after loop, if `mark_missing_gone`: `gone = await repo.mark_gone(source_id, seen_ids, now=now)`.
    - `flush`, return `SyncStats`.

### Task 5: `IngestionService` (per-source orchestration + ScrapeRun) + integration test

**Files:**
- Create: `src/realestate/ingestion/service.py`
- Test: `tests/ingestion/test_service.py`

**Interfaces:**
- Consumes: `get_scrapers`/`get_scraper`, `run_search` (Plan 2), `to_listing`, `IncrementalEngine`, `ScrapeRunRepository`, `ScraperBlocked`, `SearchCriteria`, `async_sessionmaker`.
- Produces:
  - `class IngestionService(session_factory: async_sessionmaker, fetcher)`:
    - `async def ingest(self, criteria: SearchCriteria, *, source_ids: list[str] | None = None, max_pages: int = 1) -> list[ScrapeRun]`
    - For each source (from `get_scrapers()` or given subset), in a **separate session/transaction**:
      - create `ScrapeRun(source_id, started_at=now, status=SUCCESS)`.
      - `try`: `raws = await run_search(scraper, self.fetcher, criteria, max_pages=max_pages)`; `listings = [to_listing(r, now=now) for r in raws]`; `stats = await IncrementalEngine(session).sync_source(source_id, listings, now=now, mark_missing_gone=True)`; set counters from `stats`, `status=SUCCESS`.
      - `except ScraperBlocked`: `status=BLOCKED`, `error_message=str(e)`, **without** mark_gone (sync not executed).
      - `except Exception as e`: `status=FAILED`, `error_message=str(e)`.
      - `finally`: `finished_at=now2`; save `ScrapeRun` (repo.add) and `commit`.
      - append run to result.
    - Return list of `ScrapeRun`.
