# Silnik inkrementalny + ingestia — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Połączyć scrapery z magazynem: znormalizować `RawListing`→`Listing`, inkrementalnie zsynchronizować bazę (nowe/zmienione/zniknięte oferty + historia cen) i orkiestrować przebiegi scrapingu per portal z izolacją błędów oraz statystykami (`ScrapeRun`).

**Architecture:** Czysty **normalizator** (`RawListing`→`Listing`, hash treści) + **silnik inkrementalny** (`IncrementalEngine.sync_source`) operujący na repozytoriach + **IngestionService** orkiestrujący: dla każdego źródła `run_search` (Plan 2) → normalizacja → `sync_source` → zapis `ScrapeRun`. Każde źródło w osobnej transakcji i osobnym `try` — awaria/blokada jednego portalu nie psuje pozostałych.

**Tech Stack:** Python 3.14, SQLAlchemy 2.0 async, Pydantic. Testy: pytest + realny Postgres+pgvector (testcontainers, fixture `engine`), fake fetcher + realne fixture'y HTML.

## Global Constraints

- Python 3.14; uruchamianie przez **uv**. SQLAlchemy 2.0 async. Migracje tylko Alembic.
- Normalizator jest **czystą funkcją** (`RawListing`→`Listing`), bez I/O.
- TDD; `uv run ruff check .` musi przechodzić.
- `raw_hash` = SHA-256 z kanonicznej reprezentacji znaczących pól oferty (nie z `raw`).
- Zmiana `raw_hash` → aktualizacja pól + `embedding=None` (do ponownego policzenia w Planie 4) + (jeśli zmiana ceny) wpis do `price_history`.
- Izolacja per źródło: jeden portal nie może wywrócić całego przebiegu; `ScraperBlocked` i wyjątki łapane per źródło i zapisywane w `ScrapeRun`.
- Oznaczanie „gone" tylko dla przebiegu zakończonego sukcesem (nie dla blocked/failed) — by blokada nie skasowała aktywnych ofert.

## Konwencje danych

`SyncStats` (dataclass): `new: int`, `updated: int`, `unchanged: int`, `gone: int`.

`ScrapeRunStatus` (StrEnum): `SUCCESS`, `BLOCKED`, `FAILED`.

```python
from datetime import UTC, datetime
from decimal import Decimal

from realestate.ingestion.normalize import compute_raw_hash, to_listing
from realestate.models.enums import ListingStatus, MarketType
from realestate.scrapers.base import RawListing


def _raw(**kw):
    base = dict(source_id="otodom", external_id="1", url="https://x/1", title="Mieszkanie",
                price=Decimal("600000"), area_m2=50.0, rooms=3, market="secondary",
                images=["b.jpg", "a.jpg"])
    base.update(kw)
    return RawListing(**base)


def test_to_listing_maps_fields_and_computes_ppm2():
    now = datetime.now(UTC)
    listing = to_listing(_raw(), now=now)
    assert listing.source_id == "otodom" and listing.external_id == "1"
    assert listing.market == MarketType.SECONDARY
    assert listing.price_per_m2 == Decimal("12000.00")
    assert listing.status == ListingStatus.ACTIVE
    assert listing.first_seen == now and listing.last_seen == now
    assert listing.embedding is None
    assert listing.raw_hash


def test_unknown_market_is_none():
    assert to_listing(_raw(market=None), now=datetime.now(UTC)).market is None


def test_price_per_m2_none_when_no_price():
    listing = to_listing(_raw(price=None), now=datetime.now(UTC))
    assert listing.price_per_m2 is None


def test_raw_hash_stable_and_content_sensitive():
    h1 = compute_raw_hash(_raw())
    h2 = compute_raw_hash(_raw(images=["a.jpg", "b.jpg"]))  # inna kolejność obrazów
    h3 = compute_raw_hash(_raw(price=Decimal("700000")))
    assert h1 == h2  # kolejność obrazów nieistotna (sortowane)
    assert h1 != h3  # zmiana ceny zmienia hash
```

### Task 4: Silnik inkrementalny (`IncrementalEngine.sync_source`)

**Files:**
- Create: `src/realestate/ingestion/incremental.py`
- Test: `tests/ingestion/test_incremental.py`

**Interfaces:**
- Consumes: `ListingRepository`, `Listing`, `PriceHistory`, `compute_raw_hash`-owy `raw_hash` już ustawiony w `Listing` (z normalizatora).
- Produces:
  - `@dataclass class SyncStats: new=0; updated=0; unchanged=0; gone=0`
  - `class IncrementalEngine(session: AsyncSession)`:
    - `async def sync_source(self, source_id: str, listings: list[Listing], *, now: datetime, mark_missing_gone: bool = True) -> SyncStats`
    - Dla każdego `incoming` (transient `Listing` z normalizatora, ten sam `source_id`):
      - `existing = repo.get_by_external(source_id, incoming.external_id)`
      - brak → `session.add(incoming)`; jeśli `incoming.price is not None` → dodaj `PriceHistory(price=incoming.price, observed_at=now)` powiązaną z `incoming`; `new += 1`.
      - istnieje, `existing.raw_hash == incoming.raw_hash` → `existing.last_seen = now`; `unchanged += 1`.
      - istnieje, hash różny → skopiuj pola mutowalne z `incoming` na `existing` (`title, price, price_per_m2, area_m2, rooms, floor, total_floors, city, district, street, lat, lon, market, description, images, posted_at, raw_hash`), `existing.last_seen = now`, `existing.status = ACTIVE` (reaktywacja gdy wróciła), `existing.embedding = None`; jeśli `incoming.price is not None and incoming.price != old_price` → dodaj `PriceHistory(price=incoming.price, observed_at=now, listing=existing)`; `updated += 1`.
      - zbierz `seen_ids`.
    - po pętli, jeśli `mark_missing_gone`: `gone = await repo.mark_gone(source_id, seen_ids, now=now)`.
    - `flush`, zwróć `SyncStats`.

  - oferta nieobecna w kolejnym sync → po `sync_source` z `mark_missing_gone=True` jej `status==GONE`, `gone>=1`.

- Produces:
  - `class IngestionService(session_factory: async_sessionmaker, fetcher)`:
    - `async def ingest(self, criteria: SearchCriteria, *, source_ids: list[str] | None = None, max_pages: int = 1) -> list[ScrapeRun]`
    - Dla każdego źródła (z `get_scrapers()` lub podanego podzbioru), w **osobnej sesji/transakcji**:
      - utwórz `ScrapeRun(source_id, started_at=now, status=SUCCESS)`.
      - `try`: `raws = await run_search(scraper, self.fetcher, criteria, max_pages=max_pages)`; `listings = [to_listing(r, now=now) for r in raws]`; `stats = await IncrementalEngine(session).sync_source(source_id, listings, now=now, mark_missing_gone=True)`; ustaw liczniki z `stats`, `status=SUCCESS`.
      - `except ScraperBlocked`: `status=BLOCKED`, `error_message=str(e)`, **bez** mark_gone (sync nie wykonany).
      - `except Exception as e`: `status=FAILED`, `error_message=str(e)`.
      - `finally`: `finished_at=now2`; zapisz `ScrapeRun` (repo.add) i `commit`.
      - dołącz run do wyniku.
    - Zwróć listę `ScrapeRun`.
