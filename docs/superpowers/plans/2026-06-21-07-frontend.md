### Task 2: Search page — form + results

**Components:**
- `SearchPage` — orchestrator, manages `SearchRequest` state.
- `SearchForm` — city, district, market, price/area/rooms range, sort, query (text input for vector search).
- `SearchResults` — grid of `ListingCard`.

**Data flow:**
1. User fills form, clicks "Search".
2. `POST /api/v1/search` with `SearchRequest` body.
3. Display `ListingCard` in grid (3 columns on desktop).
4. "Load more" button — fetch next page via cursor.

**Types:**
```typescript
interface ListingResponse {
  id: number; external_id: string; source_id: string;
  title: string; description: string | null; enriched_description: string | null;
  price: number | null; price_per_m2: number | null; area_m2: number | null;
  rooms: number | null; floor: number | null; total_floors: number | null;
  market: string | null; status: string; city: string | null; district: string | null;
  street: string | null; lat: number | null; lon: number | null;
  images: string[];
  url: string; posted_at: string | null;
  first_seen: string; last_seen: string;
  similarity: number | null;
}

interface SearchRequest {
  filters: { query?: string; city?: string; district?: string; market?: string;
    price_min?: number; price_max?: number; area_min?: number; area_max?: number;
    rooms_min?: number; rooms_max?: number; status?: string; };
  sort: string; sort_dir: string; limit: number; cursor?: string;
}

interface ListingsResponse { items: ListingResponse[]; next_cursor: string | null; }
```

**Tests:**
- `test_empty_form_search` — submit empty form → results fetched.
- `test_search_with_filters` — fills form fields, verifies API call payload.
- `test_load_more` — mock 2 pages, click "Load more" → second page appended.
- `test_loading_state` — shows spinner during request.
- `test_error_state` — API error → error message displayed.
