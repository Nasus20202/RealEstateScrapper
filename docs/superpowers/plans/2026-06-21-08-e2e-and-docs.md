# E2E, documentation, cleanup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** End-to-end test (docker compose + Playwright), finalize documentation, update README/diagrams, add CI config, final cleanup.

**Tech Stack:** Playwright (E2E), docker compose (full stack), GitHub Actions (CI).

## Global constraints

- Python 3.14; execution via **uv**. Docker compose for full-stack test.
- Playwright for E2E.

---

### Task 1: E2E test — full stack in docker compose

**Files:**
- `e2e/docker-compose.e2e.yml` — overrides for testing:
  - `db`: postgres + pgvector + postgis.
  - `backend`: built image or `uv run uvicorn`.
  - `frontend`: built image with `VITE_API_BASE=http://backend:8000`.
- `e2e/playwright.config.ts` — point to `http://frontend:5173`.
- `e2e/tests/search.spec.ts`:
  - Load page, expect search form.
  - Fill city = "Gdańsk", submit → wait for results.
  - Click first result → detail page loads → price, images visible.
- `e2e/tests/metrics.spec.ts`:
  - Navigate to `/metrics` → see stats cards.
- `e2e/README.md` — how to run E2E tests.

**Test (run via CI or locally):** `cd e2e && docker compose -f docker-compose.e2e.yml up -d && pnpm exec playwright test`.

### Task 4: GitHub Actions — CI pipeline

**Files:**
- `.github/workflows/ci.yml`:
  - Trigger: `push` to `main`, `pull_request`.
  - Job `backend`:
    - Setup Python 3.14 + uv.
    - `uv sync`.
    - `uv run ruff check .`.
    - `uv run pytest` (with testcontainers postgres).
  - Job `frontend`:
    - Setup Node 22 + pnpm.
    - `pnpm install`.
    - `pnpm exec vitest run`.
    - `pnpm build`.
